# KET 单词 — 产品需求文档

## 一、项目概述

面向中国孩子（10 岁以上）的 KET 英语词汇学习 Web App。目标是帮助孩子系统掌握剑桥 KET（A2 级）全部 1598 个词汇，通过每日学习 + 发音练习 + 拼写测试 + 选择题，形成记忆闭环。

- **线上地址**：https://ket-words.pages.dev
- **技术栈**：React 19 + TypeScript + Vite 8 + Tailwind CSS v4 + shadcn/ui + Zustand + React Router v6
- **数据存储**：localStorage（Zustand persist）
- **部署**：Cloudflare Pages + PWA（可离线、可添加到手机桌面）

---

## 二、词库数据

### 来源与规模
- 剑桥 KET 官方词表，共 **1598 个词条**
- 存储路径：`src/data/ket-words.json`

### 每个词条的字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | number | 唯一 ID，1–1598 |
| `word` | string | 英文单词（含消歧文本，如 `"sign (NOTICE)"`） |
| `pos` | string | 词性（英文，如 `"n"`, `"v"`, `"adj"`） |
| `pos_cn` | string | 词性（中文，如 `"名词"`, `"动词"`） |
| `definition` | string | 中文释义（简洁，如 `"能够的"`） |
| `example` | string | 英文例句（A2 难度，不超过 15 个词） |
| `example_cn` | string | 例句中文翻译 |

## 三、用户状态与进度

### 状态结构（Zustand + localStorage）

```
userId          null（未使用）
dailyGoal       每日目标单词数，默认 20，可选 10/20/30/50
currentSession  { date, wordIds[], completed }
progress        Record<wordId, WordProgress>
errorBank       number[]（错题 ID 列表）
```

### WordProgress 结构

```
wordId          单词 ID
status          'new' | 'learning' | 'mastered'
correctCount    累计答对次数
wrongCount      累计答错次数
lastStudied     最后学习时间戳
```

### 掌握规则
- 累计答对 **3 次** → 状态变为 `mastered`，不再出现在每日学习池

## 四、每日选词算法

### 难度分类（按基础单词长度，忽略消歧括号）

| 难度 | 字母数 | 示例 | 词库数量 |
|------|--------|------|---------|
| 简单 | ≤ 4 | `be`, `big`, `able` | 540 个 |
| 中等 | 5–7 | `table`, `friend`, `English` | 765 个 |
| 困难 | ≥ 8 | `beautiful`, `unfortunately` | 293 个 |

### 动态权重

根据已掌握单词数占总词库的比例 `p`（0→1），三种难度的抽取比例动态变化：

| 进度 p | 简单 | 中等 | 困难 |
|--------|------|------|------|
| 0%（刚开始） | 65% | 30% | 5% |
| 25% | 55% | 33% | 12% |
| 50% | 40% | 38% | 22% |
| 75% | 27% | 38% | 35% |
| 100%（接近完成） | 10% | 35% | 55% |

权重公式：
- `wEasy = max(10%, 65% − 55% × p)`
- `wHard = min(55%, 5% + 50% × p)`
- `wMedium = 100% − wEasy − wHard`

### 随机打乱

- 每个难度桶内部独立随机打乱（Fisher-Yates），保证每次选词不固定在同一批字母
- 从三个桶按权重取词后，再整体随机打乱，确保 session 内不是简单→中等→困难的固定顺序
- 某桶词量不足时，缺口自动由其他桶补齐

### Session 管理
- 每天第一次点"开始今日学习"时调用选词算法，将选中的 wordId 列表存入 `currentSession`
- 当天内再次进入时直接恢复同一 session（不重新选词）
- 所有单词练习完后标记 `completed: true`，当日按钮变为"今日已完成 ✓"

## 五、功能模块

### 5.1 首页（/）

- 顶部展示"已掌握 / 待学习"两个数字卡片
- 四个功能入口按钮：
  - **开始今日学习**：进入当日 session（今日完成后置灰）
  - **错题练习**：带红色角标显示错题数量（无错题时置灰）
  - **选择题练习**：随机 10 题多选练习
  - **学习统计**：进度概览 + 每日目标设置
- 底部显示当前每日目标数

---

### 5.2 今日学习（/study）

四个阶段依次进行：

#### 阶段一：展示（show）
- 显示单词、词性（中文）、中文释义、英文例句、例句翻译
- 进入页面自动播放单词 TTS 朗读（英式发音，语速 0.9）
- 右侧喇叭按钮可手动重播
- 按钮文字根据设备能力动态变化：支持语音识别显示"练习发音 →"，否则直接"开始拼写 →"

#### 阶段二：发音练习（speak）
- 仅在设备支持 Web Speech API 时显示
- 显示目标单词，提供大圆形麦克风按钮
- 录音中：按钮变红 + 脉冲动画 + 旋转加载图标
- 录音完成后：
  - 显示语音识别文字
  - 进度条显示相似度得分（0–100 分，≥ 70 分为通过）
  - 绿色进度条（通过）/ 黄色进度条（未通过）
  - 提供"再试一次"和"继续拼写"两个按钮
- 识别算法：Levenshtein 距离相似度，取 3 个候选结果中最高分
- 不支持时或点击"跳过发音"可直接进入拼写阶段

#### 阶段三：拼写测试（quiz）
- 隐藏单词，仅显示中文释义 + 例句中文翻译
- 文本框输入英文单词（大写字母不敏感）
- 支持 Enter 键提交
- "提交答案"按钮在输入为空时置灰

#### 阶段四：结果（result）
- 答对：绿色卡片 + ✅，自动播放 TTS
- 答错：红色卡片 + ❌，显示用户输入的错误内容，自动播放 TTS
- 无论对错，展示完整单词信息（单词、例句）
- 更新 progress：答对 +1 correctCount，答错 +1 wrongCount；答错时加入错题本
- 按钮：最后一词显示"完成学习 🎉"，其余显示"下一个 →"

#### 完成页面
- 显示 🎉 + "今日学习完成！" + 总词数
- "返回首页"按钮
- 触发 `completeDailySession()`（通过 useEffect，非 render 内调用）

---

### 5.3 错题练习（/error-bank）

- 从错题本取出所有单词，顺序来自加入错题本的先后顺序
- **只有拼写阶段**（无展示、无发音阶段）
- 题目：显示中文释义 + 例句中文翻译
- 答对 → 自动从错题本移除，显示绿色"已从错题本移除 ✓"
- 答错 → 留在错题本，显示红色错误提示
- 手动删除按钮（"删除错题"）：直接移除，不计入正确记录
- 错题本为空时显示 🌟 空状态页

---

### 5.4 选择题练习（/practice）

- 每次随机从未掌握单词中抽取 **10 题**（所有单词已掌握则从全库随机）
- 每题 4 个选项（1 个正确 + 3 个随机干扰项），全部从词库随机抽取
- 题目：显示中文释义 + 例句中文翻译
- 选择后立即反馈：
  - 正确选项高亮绿色
  - 错误选择高亮红色
  - 其余选项变灰
  - 显示正确单词 + TTS 自动播放
- 答错时加入错题本，同时更新 progress
- 完成页面：显示得分 + 百分比进度条 + emoji（≥80% 🏆 / ≥60% 👍 / 其余 💪）
- 支持"再练一组"（刷新页面重新随机）

---

### 5.5 学习统计（/stats）

- **总体进度条**：已掌握 / 1598
- **四格数据卡**：
  - 已掌握（绿色）
  - 学习中（黄色）
  - 错题本数量（红色）
  - 正确率 = 总答对 / 总答题（蓝色）
- **每日目标设置**：4 个快捷选项（10 / 20 / 30 / 50 个），当前值高亮

## 六、TTS（文字转语音）

- 使用浏览器原生 Web Speech API `speechSynthesis`
- 语言：`en-GB`（英式发音）
- 语速：0.9（略慢于正常速度，适合学习）
- 触发时机：进入展示阶段自动播放；结果页自动播放；各页面喇叭按钮手动触发

---

## 七、发音评估

- 使用浏览器原生 `SpeechRecognition` / `webkitSpeechRecognition`
- 语言：`en-GB`
- 最多返回 3 个候选结果（`maxAlternatives = 3`）
- 评分算法：Levenshtein 编辑距离，相似度 = `(1 − dist/maxLen) × 100`
- 取 3 个候选中与目标单词相似度最高的结果
- ≥ 70 分视为通过
- 静默无声时返回 score=0, passed=false
- 不支持时（iOS 旧版等）整个 speak 阶段不显示

## 八、UI 设计规范

- **主色**：紫罗兰 / 靛蓝（`oklch(0.58 0.22 285)`，相当于 `#7c3aed`）
- **强调色**：柔粉（`oklch(0.88 0.08 320)`）
- **字体**：Geist Variable（可变字体）
- **圆角**：基准 0.75rem，卡片用 `rounded-3xl`，按钮用 `rounded-2xl`
- **最大宽度**：`max-w-lg`（512px），居中，适配手机 / iPad 竖屏
- **深色模式**：已配置 CSS 变量，跟随系统
- 进度条颜色：主色（学习）/ 绿色（正确/通过）/ 黄色（部分正确）/ 红色（错题本）

## 九、PWA 与部署

- **Cloudflare Pages**，项目名 `ket-words`
- `public/_redirects`：所有路径回退到 `index.html`（SPA 路由支持）
- **Service Worker**：`vite-plugin-pwa` 生成，预缓存所有静态资源，离线可用
- **Web App Manifest**：
  - 名称：KET 单词 / KET单词
  - 主题色：`#7c3aed`
  - 背景色：`#faf9ff`
  - 显示模式：`standalone`
  - 方向：`portrait`
- **iOS 支持**：
  - `apple-mobile-web-app-capable`：可添加到主屏幕全屏运行
  - `apple-touch-icon`：192×192 紫色图标
  - `viewport-fit=cover`：适配刘海屏安全区
- **部署命令**：`npm run build && npx wrangler pages deploy dist --project-name ket-words --commit-dirty=true`
